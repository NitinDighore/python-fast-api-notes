# Python Web Frameworks

## What are Web Frameworks?
Developing a web application involves managing many low-level activities, such as handling network protocols, sockets, and managing processes or threads. 

A **web framework** provides a collection of predefined libraries, modules, and packages that automatically handle these low-level activities. This allows developers to focus on building the application rather than reinventing the wheel. Frameworks provide built-in tools for essential components, including:
* Form handling
* Database abstraction
* Session management
* Choosing templates for the application
* Handling requests and responses
* Applying security mechanisms

## 3-Tier Web Application Architecture
A standard web application typically follows a 3-tier architecture, where data flows from the user interface down to the database. The logic is separated into three distinct layers:

1. **Presentation Tier (Front-end)**
   * **Role:** The user interface level where users interact directly with the application (e.g., via a web browser or a mobile app).
   * **Technologies:** HTML5, CSS3, JavaScript, Angular, etc.
   
2. **Application / Service Tier (Back-end)**
   * **Role:** Often called the "logic tier" or "middle tier," this is where the core business rules, processing, and application logic are executed. It sits right between the user interface and the database.
   * **Technologies:** Python, Java, NodeJS, PHP, .NET, etc.
   
3. **Persistence Tier (Data Tier)**
   * **Role:** This is where data generated by the web application is permanently stored, managed, and retrieved. 
   * **Technologies:** Relational databases, file systems, etc.

## Types of Python Web Frameworks
Python has a rich ecosystem of web frameworks, which are broadly categorized into three types:
1. **Full Stack Frameworks** (e.g., Django)
   Provides the complete tool-kit for developing web applications including the application server, template engines, form generators, validation, authentication, DB abstraction, and so on. This category of web frameworks are heavy weight as they have predefined libraries/components that support required logic in each layer of the 3-tier web application discussed in the previous page.

2. **Lightweight / Micro Frameworks** (e.g., Flask)
   These Frameworks provide the basic components needed to start building a simple web application like handling web requests and rendering the response. Additional libraries or tools can be imported if required.

3. **Modern API Frameworks** (e.g., FastAPI)
   These frameworks are built specifically for API development (Application Programming Interface). APIs are programs that let different applications talk to each other. They are very fast and automatically check if the data you send is in the right format.

## The Need for Modern Web Frameworks
Traditional frameworks like Django and Flask are incredibly powerful and widely used. However, they were built before modern web development demands arose. 

Today's API development requires modern tools tailored for modern needs. Developers now look for frameworks that are:
* **Fast:** High performance.
* **Type-safe:** Utilizing automatic type checking to prevent bugs.
* **Async-friendly:** Built-in support for asynchronous programming.
* **Auto-documented:** Automatically generating interactive API documentation.
